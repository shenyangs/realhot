# Brand Hotspot Studio Design

## Goal

构建一个面向 AI、互联网、软件服务、SaaS 品牌团队的 Web SaaS 后台。系统基于品牌策略包，持续监测行业热点和部分大众热点，自动判断品牌相关性，设计传播角度，并生成 `2 条快反 + 2 条观点` 的图文和视频内容包，最终进入人工审核。

## MVP Scope

- 独立 Web SaaS 后台
- 图文优先覆盖小红书、微信公众号
- 视频优先覆盖视频号、抖音
- 热点监测和内容生成为真实接入边界
- 发布环节首版保留导出与排期占位
- 审核方式按热点包进行，不按单条内容拆分

## Product Modules

### Brand Brain

系统自动抓取品牌官网、公众号历史内容、客户上传知识库、最近一个月线下活动资料和媒体新闻稿，生成一份品牌策略包。策略包分成稳定层与时效层，前者包含定位、客群、语气、竞品和禁区，后者包含近期活动、媒体动作、功能更新和高管观点。

### Hot Signal Engine

系统持续采集行业热点、大众热点和品牌相关动态，并根据品牌相关性、行业重要性、传播爆发速度和风险等级综合打分，形成优先级排序。高优先级热点进入快反档，普通热点进入标准档。

### Strategy Engine

策略引擎将热点与品牌策略包做匹配，生成为什么现在发、为什么由该品牌来发、适合采用什么角度、哪些平台优先、哪些表述需要避开等结论。

### Content Studio

内容层围绕单个热点输出 4 条核心内容，分别为 `2 条快反` 与 `2 条观点`，再根据平台规范进行适配。快反内容以时效和占位为优先，观点内容以专业度和差异化表达为优先。

### Review Console

审核台以热点包为最小单位，向用户交付完整内容结果而不是先交付分析卡。用户可以审核、编辑、导出，并调整平台建议和发布时间建议。

## Technical Architecture

- Frontend: Next.js App Router
- Data layer: Supabase
- AI layer: 多模型路由层，对业务暴露统一接口
- Integration boundary: 热点采集适配器、内容生成器、发布导出器
